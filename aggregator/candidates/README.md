# Candidates

Tracking which Lean packages we are considering for inclusion in
`lean-pool`, along with the reasoning behind each decision.

## Layout

- `packages/` — one file per candidate package, holding the curated
  metadata and review status. The per-package files are the source of
  truth for everything in this directory and are populated by a
  follow-up PR.
- `raw_data/` (gitignored) — local-only Reservoir manifest snapshots.
  Running `uv run python -m lean_pool.aggregator` from `python/`
  downloads `manifest.json` here. Nothing in this folder is committed;
  if we later want a stable snapshot we will publish it as a GitHub
  Releases asset rather than tracking it in git.
