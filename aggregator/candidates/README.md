# Candidates

Tracking which Lean packages we are considering for inclusion in
`lean-pool`, along with the reasoning behind each decision.

## Layout

- `manual.txt` (committed) — hand-edited list of GitHub URLs for
  packages that Reservoir does not index (forks, missing
  `lake-manifest.json`, non-OSI license, fewer than two stars). One
  URL per line; `#` comments and blank lines allowed.
- `raw_data/` (gitignored) — local-only data fetched by
  `python -m lean_pool.aggregator fetch`: the Reservoir manifest plus
  GitHub metadata for each entry in `manual.txt`.
- The summary table below is regenerated from those files by
  `python -m lean_pool.aggregator render`. By default it shows only
  packages with at least 2 stars (matching Reservoir's own floor) so
  GitHub keeps rendering the markdown — `raw_data/` still holds every
  package, and `--min-stars=0` renders the full list. Inclusion
  decisions are not yet tracked here; when we start making them we
  will add a separate `decisions.jsonl` rather than per-package files.

Do not edit the autogenerated section by hand.
